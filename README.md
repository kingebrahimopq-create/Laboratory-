# نظام إدارة المختبر الطبي (Laboratory Management System)

نظام متكامل وحديث لإدارة المختبرات الطبية، يوفر واجهة احترافية للمالك والموظفين والمرضى.

## المميزات الرئيسية

### 🔐 نظام الأمان والمصادقة
- تسجيل دخول آمن للمالك عبر البريد الإلكتروني وكلمة المرور
- وصول المرضى عبر رقم الهاتف والاسم
- دعم Google OAuth للمالك
- حماية كاملة للبيانات الحساسة

### 👥 إدارة الموظفين والصلاحيات
- نظام أدوار مرن (مالك، طبيب، مساعد، موظف)
- إدارة الصلاحيات لكل دور
- تتبع أنشطة الموظفين

### 🧪 إدارة التحاليل
- إضافة أنواع تحاليل جديدة وتسعيرها
- نظام خصومات مرن (نسبة أو مبلغ ثابت)
- توليد رموز QR لكل تحليل
- تتبع حالة التحليل (قيد الانتظار، قيد المعالجة، مكتمل، معتمد)

### 💰 النظام المالي
- إدارة الأسعار والخصومات
- تتبع المدفوعات
- تقارير مالية تفصيلية
- إحصائيات الإيرادات

### 🖨️ دعم الطباعة
- دعم أنواع طابعات متعددة (حرارية، نافثة حبر، ليزر)
- إعدادات اتصال متعددة (USB، شبكة، بلوتوث)
- توثيق النتائج قبل الطباعة

### 📧 الإشعارات والتواصل
- إشعارات فورية للمرضى عند اكتمال النتائج
- إرسال النتائج عبر البريد الإلكتروني
- نظام إشعارات داخلي

### 📊 التقارير والإحصائيات
- لوحة تحكم شاملة بإحصائيات المعمل
- تقارير يومية وشهرية
- تحليل الأداء والإيرادات

### 🎨 واجهة عصرية وسهلة الاستخدام
- ألوان متحركة وتصاميم بسيطة
- استجابة كاملة (Responsive)
- تجربة مستخدم سلسة

## المتطلبات

- Node.js 18+
- pnpm
- PostgreSQL (عبر Supabase)

## التثبيت

1. استنساخ المستودع:
```bash
git clone https://github.com/kingebrahimopq-create/Laboratory-.git
cd Laboratory-
```

2. تثبيت الحزم:
```bash
pnpm install
```

3. إعداد متغيرات البيئة:
```bash
cp .env.example .env
# ثم قم بتعديل .env بإضافة بيانات الاتصال الصحيحة
```

4. توليد Prisma Client:
```bash
npx prisma generate
```

5. تشغيل الخادم:
```bash
pnpm dev
```

## الصفحات الرئيسية

### صفحة تسجيل الدخول (`/`)
- تبديل بين وضع المريض والمالك
- تسجيل دخول آمن
- واجهة جميلة وسهلة الاستخدام

### لوحة تحكم المالك (`/dashboard`)
- عرض إحصائيات المعمل
- إدارة المرضى والتحاليل
- إدارة الموظفين والصلاحيات
- إعدادات المعمل والألوان

### صفحة نتائج المريض (`/patient-results`)
- عرض التحاليل المكتملة
- تنزيل النتائج كـ PDF
- عرض رموز QR

## API Endpoints

### المرضى
- `POST /api/patients/search` - البحث عن مريض
- `GET /api/patients/:phone/tests` - الحصول على تحاليل المريض

### المعمل
- `GET /api/lab/stats` - الحصول على إحصائيات المعمل
- `GET /api/tests/:testId/download` - تنزيل نتيجة التحليل

## قاعدة البيانات

### الجداول الرئيسية
- `users` - المستخدمون (المالك، الموظفون)
- `labs` - بيانات المعمل
- `patients` - بيانات المرضى
- `tests` - التحاليل
- `test_types` - أنواع التحاليل
- `discounts` - الخصومات
- `printers` - الطابعات
- `appointments` - المواعيد
- `notifications` - الإشعارات

## المتغيرات البيئية المطلوبة

```
DATABASE_URL          # رابط قاعدة البيانات
DIRECT_URL            # رابط مباشر لقاعدة البيانات (للهجرات)
SUPABASE_URL          # رابط Supabase
SUPABASE_ANON_KEY     # مفتاح Supabase العام
VITE_SUPABASE_URL     # رابط Supabase للواجهة الأمامية
VITE_SUPABASE_ANON_KEY # مفتاح Supabase للواجهة الأمامية
PORT                  # منفذ الخادم (افتراضي: 3000)
NODE_ENV              # بيئة التشغيل (development/production)
```
